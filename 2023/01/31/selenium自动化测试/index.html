使用的是Python+selenium实现自动化软件测试

开始

使用selenium进行测试大致架构如下：

  • 首先selenium库中的 WebDriver通过我们的程序与本机上安装的对应浏览器版本的Driver进行交互，获得我们测试的需求
  • 然后安装的相应版本的浏览器驱动对浏览器发出动作指令，调动浏览器完成我们的相应动作

上述就是直接通信的例子，远程就是在此基础上增加了本机WebDriver和远程服务器上WebDriver的交互

元素定位

selenium提供了8个定位策略

定位器 描述
id 定位id属性与搜索值匹配的元素
name 定位name属性与搜索值匹配的元素
tag_name 定位标签名称与搜索值匹配的元素
class_name 定位class属性与搜索值匹配的元素（不允许使用复合类名）
link_text 定位link text可视文本与搜索值完全匹配的锚元素
partial_link_text 定位link text可视文本部分与搜索值部分匹配的锚点元素
XPath 定位与XPath表达式匹配的元素
css_selector 定位css选择器匹配的元素

XPath语法
XPath基本语法
  1. 层级
  2. 属性

  • / (开头): 根路径

  • / (中间): 下一级

  • // (开头,中间):任意层级

  • *任意元素

  • @ 根据属性筛选

  • . 本级

  • .. 上一级

XPath函数
  • text: 获取元素内的文本, 不光只能定位a标签文本**<精确匹配>**
  • contains: 实现任意位置的包含 <模糊匹配>
  • starts-with: 开头字符相同 <半模糊匹配>
